## CORDINGRULE



- master で作業しない。
- conflict 起こさないように master 更新されたらローカルの方も更新する。
- 自分が変更したファイル以外は極力上げない。
- ブランチ名はキャメルケースを使う。
- コミット名は日本語で何をしたのかわかりやすくつける。
- Pull Request(PR)と Issue を関連づける。
- 変数名はちゃんとして("a"とか"b"はダメ）

### ブランチ名

lowerキャメルケースで記述

ex)

```
makeHeader
```

### ファイル名
スネークケースで記述

ex)

```
data_transfer_page.dart
```
